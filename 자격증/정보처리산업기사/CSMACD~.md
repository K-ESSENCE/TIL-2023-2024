### CSMACD

CSMA/CD 방식

CSMA 방식에서와 같은 충돌이 발생하는 문제점을 해소하기위해 CSMA 방식에 충돌 검출 기능과 충돌 발생 시 재송신하는 기능을 부가

통신하기전에 누가 이 통신회선 쓰고있는지 확인

Carrier Sense => 통신회선이 다른애들이쓰고있는지 아닌지점검하는것

Multiple Access => 여러개가 액세스하면 충돌이 발생하면 데이터가 정상적으로 전달이 안됨

Collision Detection => 충돌여부 확인

이런문제점을 해결하기위해 CSMA에 충돌검출과 충돌시 재송신하는 기능부가한것 => CMSACD.

CSMA CD방식은 통신회선이 사용중이면 대기하고, 데이터가 없을떄에만 데이터를 송신.

송신중에도 전송로의 상태를 계속 감시.

충돌이 발생하면 송신을 중지 후 모든 노드에 충돌을 알린 후 일정 시간이 지난 다음 데이터를 재송신함.

성형 또는 버스형 LAN에서 가장 일반적으로 사용됨.

전송량이 적을떄 매우 효율적이며 신뢰성이 높다

### 네트워크 관련 장비

한 사무실이나 가까운 거리에 있는 컴퓨터들을 연결해 주는 장치.

각 회선을 통합적으로 관리하며, 신호 증폭 기능의 리피터 역할도 포함.
허브의 종류에는 더미 허브, 스위칭 허브가 있음.
전부다 공유기에 연결하기 힘드니 허브라는 장비 사용

더미허브 ? => 네트워크에 흐르는 모든 데이터를 단순하게 연결만 함.

들어온 컴퓨터 5개가있으면 5개 전부다 데이터 날림 누가 누군지모르기 때문
LAN이 보유한 대역폭을 컴퓨터 수만큼 나누어 제공.

통신하는애도있고 안하는애도 있고 섞여있음에도 불구하고 대역폭을 사용하게 됨.

네트워크에 연결된 각 노드를 물리적인 성형 구조로 연결.

스위칭 허브? => 네트워크 상에 흐르는 데이터의 유무 및 흐름을 제어해서 각각의 노드가 허브의 최대 대역폭을 사용할수있는 지능형 허브

얘는 맥 주소를 인식해서 누가누구인지 구분을함.

최근 사용되는 허브는 대부분 스위칭 허브.

리피터는 증폭기 => 전송되는 신호가 전송 선로의 특성 및 외부 충격 등의 요인으로 인해 원래의 형태와 다르게 왜곡되거나 약해질 경우 원래의 신호 형태로 재생하여 다시 전송하는 역할을 수행함.

다 날라가지못하는 것을 방지하기위해서. 리피터를 사용

근접한 네트워크 사이에 신호를 전송하는 역할로 전송거리의 연장이나 배선의 자유도를 높이기 위한 용도로 사용.
OSI참조 모델중 물리 계층에 해당함.

  물리계층 -> 허브, 리피터

  네트워크 계층 - 라우터

  데이터 링크 계층 중 MAC계층에서 사용됨으로 MAC브리지라고도한다.
  각 부서들을 서로서로 연결할때 쓰는 장비.
  네트워크 상의 많은 단말기들에 의해 발생되는 트래픽 병목 현상을 줄일수있다.
  네트워크를 분산적으로 구성할 수 있어 보안성을 높일 수 있다.

브리지를 이용한 서브넷 구성 시 전송 가능한 회선수는 브리지가 n개일때 n(n-1)/2 개임

두계의 컴퓨터상 연결 유지 (흐름/에러제어)는 LLC (로지컬 링크 계층)

### 인터넷 주소 체계

IP주소는 인터넷에 연결된 모든 컴퓨터 자원을 구분하기 위한 고유한 주소.

IPv4 라는 아이는 총 32비트로 구성되어있음.

총 32개쓰면 그게 ipv4임

가장 작은 주소는 0을 32개쓰면 가장 작은 ipv4주소가됨

가장 큰건? => 1 32개

약 42억개 가량이있음.

8/ 8/ 8/ 8

8비트씩 잘라서 32비트로 구성 중간중간 '.'을 찍어서 구분
192.168.10.0=>쓰레기주소

ip주소가 나오면 반드시 서브넷 마스크가 나와야됨(SM)

서브넷 마스크? => 쓸수있는거 없는거 구분해서 주는 거

1111111 쭉가다가마지막이 다 0 이면 거기만 맘대로 바꿔서쓰라는소리

0~255

마지막꺼면 0이면 나는 256개의 주소를 쓸 수있다.

/24 ? => 서브넷 마스크의 1의갯수가 32개중 24

192.128.0.0/24 //앞에있는 24개빼고 나머지만 쓰란 소리

255.255.255.0 이렇게도 표현가능

16면 두개 막아둔거 / 8 이면 하나
막아놓은 주소를 네트워크 주소라고하고 내가쓸수있는것을 호스트 주소라고함.

네트워크 부분의 길이에 따라 A클래스에서 E클래스까지 총 5단계로 구성되어있음.date
A는 하나 B는 두개 이렇게 증가
D클래스는 멀티캐스트용으로 224~239
E클래스는 실험적 주소 아예 일반적으로는 안씀

A는 국가나 대형 통신망 사용
B는 중대형 통신망 C는 소규모 통신망

IPv6 는 현재 Ipv4의 주소 부족 문제를 해결하기 위해 개발됨

v4에 비해 자료 전송 속도가 빠름.

인증성, 기밀성, 데이터 무결성 지원으로 보안 문제를 해결 가능

ipv4는 32비트인데 ipv6는 128비트

주소의 확장성, 융통성, 연동성이 뛰어나고 실시간 흐름 제어로 향상된 멀티미디어 기능 지원

트래픽 클래스와 Flow Label을 이용하여 등급, 서비스 별 패킷 구분가능 => 품질 보장 용이

패킷 크기를 확장할 수 있으므로 패킷 크기에 제한이 없음

기본 헤더 뒤에 확장 헤더를 더함으로 더욱 다양한 정보의 저장이 가능해져 네트워크 기능 확장이 용이

미리 예약된 알고리즘을 통해 고유성이 보장된 주소를 자동으로 구성. => 자동으로 네트워크 환경 구성이 가능

헤더 : 패킷 전송 시 제일 앞에 배치되는 40Byte 옥텟 크기의 영역으로 주소의 버전 인증정보 패킷에 대한 정보 출발 도착 주소 등의 다양한 정보를 포함함.

### 소프트웨어 공학

소프트웨어 공학은 소프트웨어의 위기를 극복하기 위한 방안으로 연구된 학문

소프트웨어 위기현상? => 소프트웨어 개발 속도가 하드웨어 개발 속도를 따라가지못해 사용자들의 요구사항을 처리할 수 없는문제 발생하는것

1. 개발 인력의 부족과 그로 인한 인건비 상승
2. 성능 및 신뢰성 부족
3. 개발 기간의 지연 및 개발 비용의 증가
4. 유지보수의 어려움과 이에 따른 비용 증가
5. 소프트웨어의 생산성과 품질 저하

IEEE 소프트웨어 공학 표준에서도 소프트웨어 개발, 운용, 유지보수, 폐기 처분에 대한 체계적인 접근 방안이라고 정의함

- 소프트웨어 공학의 기본 원칙
  현대적인 프로그래밍 기술을 계속적으로 적용해야됨
  개발된 소프트웨어의 품질이 유지되도록 지속적으로 검증해야됨
  소프트웨어 개발 관련 사항 및 결과에 대한 명확한 기록을 유지해야됨
