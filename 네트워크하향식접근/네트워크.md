# 01 컴퓨터 네트워크의 종류

## 컴퓨터와 컴퓨터를 연결하는 컴퓨터 네트워크

컴퓨터 네트워크는 여러 대의 컴퓨터를 서로 여녈하여 서로 데이터를 주고받을 수 있도록 망이 사전에 구축된 것을 말한다.

연결되는 컴퓨터의 대수가 많아질수록 네트워크의 규모는 커지고, 규모가 커진 만큼 더 다양한 정보를 교환하거나 통신 방식을 사용할 수 있게 된다.

근거리 위치 ⇒ LAN

비교적 먼거리 위치 ⇒ WAN

## 휴대전화나 스마트폰을 연결하는 모바일 네트워크

휴대전화나 무선 기술을 활용한 것을 무선 네트워크 혹은 모바일 네트워크라 부른다.

무선 네트워크의 비중이 유선 네트워크 보다 크지 않지만 오늘날에컨텐츠 전송

메일 서버는 메일 송수신

## 피어 투 피어

네트워크에 연결된 두 대의 컴퓨터가 클라이언트와 서버의 역할을 동시에 할 수있어 서비스를 주거나 받을 수 있는 통신 방식을 p2p 피어투 피어 라고 한다.

주로 개인 컴퓨터간의 파일공유나 인터넷 전화등에 활용된다.

# 04 패킷교환방식

컴퓨터 네트워크에서는 이메일이나 파일과 같은 데이터를 패킷이라는 작은 단위로 분할 한 후 주고 받는다. 는 무선으로 접속하는 장비의 대수가 많아지고 규모도 점점 커지고 있다.

# 02 컴퓨터 네트워크의 역할

실시간 커뮤니케이션을 위해 채팅이나 VoIP(인터넷 전화)를 사용한다

분산처리 ⇒ 복잡한 계사놔 처리들을 여러 대의 컴퓨터에 나누어 병렬 처리한다.

# 03 서버와 클라이언트

웹 서버는 웹페이지

패킷은 자신이 어디로 전달되어야 하는지 알 수 있도록 어드레스 정보를 가지고있다.

## 회선 교환 방식과 패킷 교환 방식

아날로그나 3g 는 회선 교환 방식을 사용하낟.

패킷 교환 방식은 주고받ㄷ을 데이터를 작게 쪼갠뒤 통신 경로를 공유하며 전송하는 방식이라 여러 상대와 통신할 떄 효과적임

회선 교환방시은 일대일이고 속도가 안정적임

회선 교환은 통로 점유하고 패킷은 한통로에 여러개가 와리가리

주고받는 패킷이 많아지면 전송 속도가 느려질 수 있음 / 메일 발송과 웹 서핑을 동시에 할 수있음

# 05 컴퓨터 네트워크와 계층 모델

통신에 참여하는 여러 장비나 프로그램들은 각각 맡은 역할이 있는 이 역할들을 이해하기 쉽게 분류하고 추상화한 것을 계층 모델이라고 한다.

앱 계층 - 웹 서비스, 이메일과 같은 서비스 제공 / 서버 클라이언트 http, smpt, pop3 , ftp ,ssh

트랜스포트 계층 - 데이터가 올바르게 전달되게 중계 / tcp , udp

인터넷 계층 - 목적지의 Ip 어드레스로 데이터 전달 / ipv4 ipv6 icmp 라우팅 등

네트워크 인터페이스 계층 - 네트워크 어댑터와 같은 하드웨어를 통해 데이터 전달 / 이더넷 무선랜 mac 어드래서 등

서비스의 내용을 결정하는 것은 앱계층 말고 없음 나머지 3계는 데이터 전달 통신기능

### 오픈 마켓과 계층 모델

홈 쇼핑이나 오픈 마켓에서도 상품이 배송되는 형태나 과정을 게층 모델로 표현할 수 있음.

### 각 계층을 통과하는 데이터의 형태

- 송신지

앱 계층 ⇒ 한 계층만 보면 같은 계층끼리 데이터를 주고받은 것처럼 보인다.

트랜스포트 계층 → 전송하기에 적합한 크기로 작게 쪼개고 식별 정보 덧붙임

인터넷 계층 → 수신 측의 컴퓨터 식별 정보

네트워크 인터페이스 → 하드웨어에 신호를 전달하는 데 필요한정보

하드웨어 → 전기나 광신호로 변환

- 하드웨어를 통과하는 데이터의 형태

네트워크 인터페이스 헤더 ⇒ 유선 lan에서 데이터를 보내는데 필요한 정보

인터넷 계층 헤더 ⇒ 목적지 식별 번호

트랜스포트 계층 헤더 → 결합 순서와 받을 프로그램 식별 신호

패킷 몸동 (분할된 데이터)

다시 네트워크 인터페이스 트레일러

데이터 앞에 붙이는 정보를 헤더라고 하고 뒤에 붙이는 걸 트레일러라고함.

- 수신지

네트워크 인터페이스 계층 → 헤더 트레일러 떼고 인터넷으로

인터넷 계층 - 수신해야된다고 판단하면 트랜스포트로

트랜스포트 → 모든 데이터 결합되면 프로그램에게

네트워크쪽 헤더 : 유선 lan 데이터 보내는 데 필요한 정보 / 인터넷 헤더 : 목적지 식별 / 전송 헤더 : 프로그램 결합순서랑 식별/ 패킷/ 네트워크 트레일러

그리고 순서대로 분리 파바박

# 06 4개 계층의 동작방식

### 애플리케이션 계층

한 계층만 들여다 보면 마치 송신과 수신측이 직접 통신하는것처럼보임

웹브라우저와 웹서버 간의 통신

애플리케이션 계층의 프로그램은 데이터의 내용을 보고 그에 맞는 서비스를 처리하는 데 집중하고 데이터 전송에 대해서는 하위 계층에 위임한다.

### 트랜스포트 계층

프로그램에서 전달받은 데이터를 목적지 애플리케이션 계층의프로그램까지 전달하는것

**재전송도 이 계층이 하는일**

1. 어떤 프로그램들이 서로 통신을 해야하는지에 대한 정보를 헤더에 기록함
   1. 표시하고 싶은 url은 ~이다
   2. 웹브라우저 ~포트에서 서버 80으로 요청을 보낸다.
   3. 받는 쪽은 포트번호를 확인하고 데이터 전달.

트랜스포트 계층은 목적지 컴퓨터의 프로그램에 데이터가 제대로 전달되도록 포트 번호 정보를 헤더에 붙이거나 뗴어내는 일을 주로 함.

### 인터넷계층

데이터에 주소 정보를 덧부여 목적지까지 무사히 전달하는 것. 인터넷 통신에서는 목적지 컴퓨터까지 도달할 수 있는 경로를 찾아야 하는데, 이떄 라우터라는 장비가 사용됨.

트랜스포트 헤더 + 인터넷계층 헤더 : 클라이언트 PC(주소)에서 웹서버(주소)로 요청 한다

어드레스 정보를 보고 목적지로 데이터 전달.

자신이 수신해야 하는 데이터라고 판단하면 트랜스포트 계층에 데이터를 전달

인터넷 통신을 할 때는 이어달리기를 하는 것처럼 데이터를 다음 네트워크로 전달하기 위해 라우터라는 장비를 사용함.

### 네트워크 인터페이스 계층

유선 LAN이나 무선 LAN 어댑터가 처리할 수 이는 형태로 데이터를 변환하고 이 데이터를 목적지까지 전달.

1. 주변장치까지 데이터를 전달하는 데 필요한 정보를 덧붙임
2. 자신이 수신해야할 데이터라고 판단하면 인터넷 계층에 넘김
3. 주변 장치까지 데이터를 전달하는데 필요한 정보를 덧붙임
4. 2와 같은 과정

네트워크 어댑터가 처리할 수 있는 형태로 데이터를 변환하고 인접한 장비나 컴퓨터까지 변환된 데이터를 전달함.
